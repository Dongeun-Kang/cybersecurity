# PortSwigger — OS Command Injection Labs Write-up

## Lab 1 — OS command injection, simple case

### Description

The application executes a system command using user-supplied input without proper validation. This allows additional commands to be appended and executed by the operating system.

### Exploit Method

```text
||whoami||
```

The payload uses a command separator to execute an additional system command after the intended command.

## Lab 2 — Blind OS command injection

### Description

The application executes a system command using user-supplied input but does not return the command output in the HTTP response. This creates a blind command injection scenario where command execution must be confirmed indirectly.

The input field expects an email address. By supplying an invalid value such as `x`, the application attempts to process the request but fails to find a matching email or complete the intended operation.

### Exploit Method

```text
email=x|| ping -c 10 127.0.0.1 ||
```

The value `x` is used because it typically does not represent a valid email address in the system. As a result, the original command fails.

The logical OR operator (`||`) instructs the shell to execute the next command only if the preceding command fails. When the first command fails due to the invalid input, the `ping` command is executed.

The repeated network requests generated by `ping` introduce a noticeable delay in the server response, confirming that command execution occurred.

## Lab 3 — Blind OS command injection with output redirection

### Description

The application executes user input within a system command but does not display the output in the HTTP response. However, attackers can redirect command output to a file stored within a web-accessible directory.

### Exploit Method

```text
||whoami>/var/www/images/output.txt||
```

The payload redirects the output of the `whoami` command to a file located in a directory accessible through the web server. The result can then be retrieved by requesting the file directly through the browser.

---

## Key Learning Points

* Blind command injection occurs when command execution does not produce visible output.
* Invalid input can be intentionally used to trigger command failure conditions.
* The `||` operator allows execution of a secondary command when the primary command fails.
* Output redirection enables retrieval of command results through accessible files.
